Software Testing:
  • Glass Box Testing
  • Black Box Testing
  • Beta Testing
  • Alpha Testing
     
  • Software tester is aware of the interior structure of the software and uses this knowledge when designing the test.
  • Tests that do not rely on knowledge of the software’s interior composition.
  • Preliminary version of the software is given to a segment of the intended audience with the goal of learning how the software performs
    in real-life situations before the final version of the product is solidified and released to the market.
  • User testing performed at the developer's site.
Documentation:
  • User Documentation
  • System Documentation
  • Technical Documentation
     
  • Explain the features of the software and describe how to use them.
  • Describe the software’s internal composition so that the software can be maintained later in its life cycle.
  • Describe how a software system should be installed and serviced.
