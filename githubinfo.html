Q: What is a version control system?

A: A category of software tools that helps in recording changes made to files by keeping track of modifications done in the code.

Q: What is Git?

A: A free and open source version control system used to handle projects effeciently. It is used to track changes in source code, which enables multiple developers to work together.

Q: What is GitHub?

A: It is a code hosting platform for collaboration and version control. You can host and review code, manage projects, and build software.

Q: What is the difference between Git and GitHub?

A: Git is a version control system that lets you manage and keep track of source code history. GitHub is a cloud-based hosting service used to manage Git repositories.

Q: Who started GitHub and how was it started?

A: Tom Preston-Werner, Chris Wanstrath, P.J. Hyett, and Scott Chacon started GitHub in 2008. It was started from these founders as a start up until 2012 when Adreessen Horowitz invested $100 million in venture capital.

Q: What company owns it now?

A: Microsoft

Q: How much does a GitHub account Cost?

Git/GitHub terms:

  • repository - collection of files of a project.
  • commit - a revision or change to files(s). A commit creates a unique identifier that allows for record keeping of changes to files and who made the changes.
  • fork - copy of a repository.
  • push - updates the remote branch with localk commits. It is one of the four commands that prompts interaction with the remote repository.
  • pull requests - lets you tell others about changes you have pushed to a branch in a repository.
  • workflows - configurable automated process that will run one or more jobs.
  • issues - tracking tool that is integrated with the repository.
  • raw button - opens the file in raw form without any html formatting.
  • blame button - used to examine the contents of a file line by line and see when each line was modified and who the author of the modification.
